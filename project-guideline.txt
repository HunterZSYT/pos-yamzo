I have made a repo at [HunterZSYT/pos-yamzo](https://github.com/HunterZSYT/pos-yamzo). Connect to this repo. We are building a local Windows POS system for Yamzo restaurant.

Main goal:
Build a simple, local, installable Windows POS app. I will first test it on my laptop. Later, I should be able to push it to GitHub, then download/clone it on the restaurant POS computer and install/run it locally.

Use a local-first setup:

* No cloud database for now.
* Use SQLite locally.
* Store database, settings, logo, QR, tokens, and printer config on the local computer.
* Do not commit local database, credentials, tokens, or secrets to GitHub.

Preferred stack:
Use Electron + React + Node.js + SQLite unless there is a strong reason to choose something else. The app should be packageable as a Windows installer/exe.

Important local files:

* Menu CSV path: `E:\Yamzo\POS\Yamzo - menu old.csv`
* Google/Gmail credential notes path: `E:\Yamzo\POS\yamzo_google_creds.txt`
* Do not commit these files or any secrets.

Printer:

* Printer: Xprinter XP-80T thermal receipt printer
* Paper: 80mm thermal paper
* Connection: currently connected to laptop
* Supports ESC/POS
* Add printer settings and test print option.
* Support USB/Windows printer flow first. Bluetooth can be optional.

Login:
Create local login.
Default:

* Username: `admin`
* Password: `1234`

Admin must be able to change password later.
Store password hashed, not plain text.

UI priority:
The POS will be used by normal restaurant staff, so the UI must be very simple.
Design for rush-hour use:

* Big buttons
* Clear labels
* Few clicks
* No complicated screens
* Current order/cart always visible
* Admin/settings separate from normal cashier screen
* Confirm dangerous actions like delete/void/cancel

Daily staff flow:

1. Login
2. New Order
3. Choose dine-in / takeaway / parcel / delivery / external manual order
4. Add items
5. Send to Kitchen if needed
6. Add more items later if needed
7. Apply discount if needed
8. Settle payment
9. Print final receipt

Restaurant order behavior:
This is not a KFC-style upfront payment system. It must support running/open restaurant orders.

Order requirements:

* Create open order
* Dine-in table number
* Takeaway/parcel/delivery
* Add items
* Add item/order notes
* Send items to kitchen
* Add more items later
* Print only newly added items as addition KOT
* Void/cancel items with reason
* Apply discount
* Settle payment
* Print final receipt
* Reprint receipt
* Keep full order history

External/manual orders:
Yamzo receives orders from Foodpanda, Foodie, and other external sources. These are not connected to the POS. At the end of shift, staff will manually enter those orders.

Add order source support:

* In-house
* Takeaway
* Parcel
* Delivery
* Foodpanda
* Foodie
* Other

External/manual orders should:

* Be quick to enter
* Not require kitchen print
* Appear in order history
* Appear in sales reports
* Be separated in analytics by source
